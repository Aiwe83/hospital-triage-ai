<ai_visual_observability>
Each visual agent represents the operational state of an autonomous entity in the system. The visual representation enables real-time human supervision, task traceability and early detection of failures or inefficient resource usage.

This is not a game-like decoration. It is AI visual observability.

The frontend must help the operator detect:
- blocked agents
- repeated prompts
- agents waiting for context
- workflow loops
- excessive LLM calls
- token/resource limit issues
- bottlenecks between specialized agents

Visual signal mapping:
- static/idle agent: possible waiting or blocked state
- looping movement: possible prompt repetition or workflow loop
- too many agents in one area: bottleneck
- red agent: error, tool failure or token limit
- yellow agent: waiting for response, context or external resource
- green agent: task completed successfully

Design goal:
Make the multi-agent AI workflow observable, understandable and supervisable through the visual layer.
Connect the visual states with backend/LangGraph events wherever possible.
Track agent steps, waiting states, errors, completions and iteration counts in agent_trace.
Avoid unnecessary LLM calls and detect repeated loops or excessive iterations.
</ai_visual_observability>
